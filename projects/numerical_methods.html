Numerical Methods in Julia

In algebra, we study methods to solve equations like 4x2+2x=04x2+2x=0 for an unknown variable xx. Differential Equations, on the other hand, we study methods to solve equations like 4y”+2y′+y=04y”+2y′+y=0 for some unknown function y=ϕ(x)y=ϕ(x). There are several ways to can express a solution in differential equations - explicitly, implicitly, and numerically.

If an explicit solution, i.e. y=ϕ(x)y=ϕ(x), cannot be found, we may be able to find an implicit solution, that is, a relationship or expression H(x,y)=0H(x,y)=0 that defines a solution ϕϕ implicitly. However, if the solution cannot be expressed implicitly or explicitly, we can formulate a numerical approximation that solves the equation. These techniques are called numerical methods. Numerical methods rely on iteration to arrive at a solution.

To understand the intuition of numerical methods and approximation, let us try to approximate the solution to the equation f(x)=x2−3x+1=0f(x)=x2−3x+1=0 using fixed point iteration. We approximate the solution by rewriting the equation in terms of xx resulting in the following: g1(x)=(x2+1)3g1(x)=(x2+1)3 and g2(x)=3−1xg2(x)=3−1x. We then iterate over these two equations until they converge.

module Fixed_Point_Iteration
function g1(start,iters)
    g1(x) = (x^2+1)/3
    x = zeros(iters)
    x[1] = start
    for i in 2:iters
        x[i] = g1(x[i-1])
    end
   return x
end
function g2(start,iters)
    g2(x) = 3-1/x
    x = zeros(iters)
    x[1] = start
    for i in 2:iters
        x[i] = g2(x[i-1])
    end
   return x
end
end

Euler’s Method

Now, lets apply the same intuition above to approximate the solution to a differential equation. Suppose we want to approximate the solution of y′=f(x,y)y′=f(x,y) with an initial value of y(x0)=y0y(x0)=y0. Around 1700s, a German mathematician named Leonhard Euler, discovered that if we pick some interval II starting at the initial condition and draw a tangent line, the points are close to the solution. If we divide interval II into smaller steps hh, we could approximate all points along the solution curve.

Euler’s method of approximation can be summarized as:

xn+1=xn+hxn+1=xn+h

yn+1=yn+hf(xn,yn)yn+1=yn+hf(xn,yn)

Around the late 1800s, another German mathematician Karl Heun, discovered if we were to take in more values into account and use a predictor-corrector function, we could obtain a better solution. The Heun method is defined as:

yn+1=yn+12h[f(xn,yn)+f(xn,yn+hf(xn,yn))]yn+1=yn+12h[f(xn,yn)+f(xn,yn+hf(xn,yn))]

using Statistics
function MAPE(y,y_n)
        return (mean(broadcast(abs,((y - y_n ) ./ y)))*100)
    end
function euler(x,y,y_n)
    h = x[2]-x[1]
    for i in 1:length(x)-1
        y_n[1] = x[1]
        y_n[i+1] = y_n[i] + h*(2*(x[i]*y_n[i]))
    end
    println("True Values: ",y)
    println("Approximate: ",y_n)
    println("MAPE: ",MAPE(y,y_n),"%")
end
function euler_improved(x,y,y_n)
    h = x[2]-x[1]
    for i in 1:length(x)-1
        y_n[1] = x[1]
        k1 = h*(2*(x[i]*y_n[i]))
        k2 = h*(2*(x[i+1]*y_n[i]+k1))
        y_n[i+1] = y_n[i] + .5*(k1+k2)
    end
    println("True Values: ",y)
    println("Approximate: ",y_n)
    println("MAPE: ",MAPE(y,y_n),"%")
end

Runge-Kutta Method

Around the early 1900’s, two German mathematicians, Carl Runge and Wilheim Kutta published their thesis Classical Runge-Kutta method of fourth order outlining a numerical approach to approximating first order differential equations. Their method uses a weighted sum of slopes at the mid point between each step hh

function runge_kutta(x,y,y_n)
    h = x[2]-x[1]
    for i in 1:length(x)-1
        y_n[1] = x[1]
        k1 = h*(2*(x[i]*y_n[i]))
        k2 = h*(2*(x[i]+.5*h)*(y_n[i]+.5*k1))
        k3 = h*(2*(x[i]+.5*h)*(y_n[i]+.5*k2))
        k4 = h*(2*(x[i]+h)*(y_n[i]+k3))
        y_n[i+1] = y_n[i] + ((k1+(2*k2)+(2*k3)+k4)/6)
    end
    println("True Values: ",y)
    println("Approximate: ",y_n)
    println("MAPE: ",MAPE(y,y_n),"%")
end
